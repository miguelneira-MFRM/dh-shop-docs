§ 02 — WorkflowQuick start

  1. Export repomix from the Matcher master app repo (see config/repomix.config.json)
  2. Save to inbox/repomix-YYYY-MM-DD.xml
  3. Eyeball the healthy export check below
  4. In Cursor: invoke the sync skill — "Sync context from latest repomix"
  5. Review context/CHANGELOG.md and update overlays/ if behavior changed

Cursor invocation (primary)

In this project's Cursor window, start a new chat and say exactly:

Sync context from latest repomix

Manual fallback

cd c:\MFRM\Cursor\matcher-knowledge-hub
npm run sync
# or with an explicit file:
npm run sync -- inbox/repomix-2026-06-03.xml

Step-by-step:

npx tsx scripts/parse-repomix.ts inbox/repomix-YYYY-MM-DD.xml
npx tsx scripts/diff-manifest.ts inbox/repomix-YYYY-MM-DD.xml
npx tsx scripts/generate-context.ts inbox/repomix-YYYY-MM-DD.xml

Then review context/CHANGELOG.md and update overlays/ for any flagged buckets.

§ 03 — PublishingPublishing plan docs as HTML

Internal Markdown docs can be published as polished HTML for outside teams. HTML is agent-generated from the design system in .cursor/skills/convert-techdoc-html/references/ — edit Markdown, not HTML, for content changes.

  1. Write or update Markdown in docs/plans/ (or a listed root doc like docs/matcher-api-triggers.md)
  2. In Cursor: "Convert tech doc to HTML" — optionally name the file
  3. HTML lands in docs/html/{basename}.html
  4. Optional sanity check: npm run check-techdocs

Cursor invocation

Convert tech doc to HTML

§ 04 — Quality gateHealthy export check

Before running sync, confirm the export looks right (calibrated from 2026-06-03):

CheckExpected
Total <file> entries in repomix XML~2,100 (full master-repo export)
Routed into context buckets~850
Key paths presentsrc/pages/index.tsx, src/pages/results.tsx, src/pages/api/, src/utils/matches/, testSetup/
Key paths absentnode_modules/, .env, dist/, .next/
Filenamerepomix-YYYY-MM-DD.xml in inbox/

If total file count swings >20% sprint-over-sprint with no known scope change, stop — repomix config drift, not code change.

Unrouted files (~1,200) are expected: infra, configs, and app paths outside the 7 buckets land in reports/unrouted-files.json.
